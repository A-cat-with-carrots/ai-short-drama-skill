# 阶段 2：故事架构（节奏地图）深度细则

## 节奏类型表

每个 beat 必属于一种类型。在 `beat_sheet.json` 用这个枚举：

| 类型 | 含义 | 持续 | 何时用 |
|------|------|------|--------|
| `cold_open` | 黄金 3 秒钩子 | 0-3s | 每集开头第 1 秒 |
| `setup` | 交代背景 | 3-15s | 仅前 3 集 / 新角色登场时 |
| `small_climax` | 小高潮 | 1-3s | 每 15-20s 一次 |
| `twist` | 反转 | 1-2s | 每 30-60s 一次 |
| `satisfaction` | 爽点（公式触发） | 2-5s | 每 30-60s 一次 |
| `pain` | 虐点 | 2-5s | 虐剧每集 1-2 次 |
| `reveal` | 揭秘（信息释放） | 1-3s | 关键节点 |
| `setup_payoff` | 兑现伏笔 | 1-3s | 第 N 集兑现第 M 集埋的 |
| `cliffhanger` | 卡点钩子（结尾） | 2-5s | 每集最后 5 秒必有 |
| `paywall_punch` | 付费点强反转 | 3-5s | 付费短剧第 10/15/20... 集结尾 |

## 节奏铁律（每集自查）

1. **第 1 秒就要有动作 / 冲突 / 反差**，不能是字幕、不能是平静镜头
2. 整集时长内，平均 **每 20s 至少 1 个 small_climax 或 twist**
3. 整集时长内，**至少 1 个 satisfaction**（爽剧 ≥ 2 个）
4. 结尾必有 **cliffhanger**，且要"够狠"（足以让人秒点下一集）
5. 不能连续 3 集都是同一种节奏类型作主导（会单调）

## N 集模板

### 24 集免费版

| 集数 | 阶段 | 任务 | 关键 beat |
|------|------|------|-----------|
| 1-2 | 钩子起锚 | 抛核心矛盾 | cold_open + 一次大 reveal + cliffhanger |
| 3-5 | 角色立住 | 立人物关系 | satisfaction + small_climax 高频 |
| 6-10 | 矛盾升级 | 两难选择、误会 | pain + twist 增多 |
| 11-15 | 高潮前夜 | 真相浮现、关键反转 | reveal + paywall_punch（如付费）|
| 16-20 | 情绪爆发 | 大冲突、生死关 | pain + satisfaction 极化 |
| 21-23 | 兑现伏笔 | 收线、清账 | setup_payoff 密集 |
| 24 | 结局 | 团圆 / 余韵 | satisfaction 收尾，留 1 钩子（续集） |

### 60 集付费版

| 集数 | 阶段 | 付费点 |
|------|------|--------|
| 1-9 | 免费抓人 | 第 9 集结尾 paywall_punch（决定第 10 集付不付费） |
| 10 | 💰首付费 | 兑现第 9 集反转 + 立刻埋新钩子 |
| 10-14 | 第一付费段 | 第 14 集 paywall_punch |
| 15-19 | 第二付费段 | 第 19 集 paywall_punch |
| 20-29 | 中段冲突 | 每 5 集 paywall_punch |
| 30-39 | 转折段 | 大反转 + 揭秘 |
| 40-49 | 高潮段 | 情绪极化 |
| 50-58 | 收线段 | 兑现所有伏笔 |
| 59-60 | 结局 | 钉死 |

### 100 集付费版

在 60 集模板基础上：
- 1-9 钩子 → 1-10 钩子（多一集铺垫）
- 中段冲突段拉长到 30-59
- 付费点节奏：第 10 集首付，之后每 5 集一付，共 18 个付费点

## 钩子怎么写（每集 cold_open 和 cliffhanger）

### cold_open（前 3 秒）

5 种万能开场（任选）：

1. **悬念问句**：黑屏 + 一句独白 → 切镜头
   - 例："你知道吗，我老公昨天死了。但他今天回来了。"

2. **极端动作**：直接砸 / 摔 / 撞 / 扇耳光
   - 例：女主一个耳光打飞男人，男人撞翻一桌酒席

3. **反差画面**：身份与场景极不匹配
   - 例：豪车里的西装男 + 身后是猪圈

4. **意外反转**：3 秒内一个反转
   - 例：女主跪地求婆婆 → 婆婆笑着扶起 → 镜头一转婆婆已经下毒

5. **直白宣战**：角色第一句话就是宣战
   - 例："苏曼，今天这场离婚官司，你输定了。"

### cliffhanger（每集最后 5 秒）

3 种万能卡点：

1. **真相揭露半句**：眼看要说出真相 → 黑屏
   - 例："其实你不是我亲生的——" 黑屏

2. **危机即将发生**：危险已经来 → 黑屏
   - 例：刺客举刀刺向女主 → 黑屏

3. **新人物登场**：陌生人推门 → 镜头给脸 → 黑屏
   - 例：女主以为已死的前夫站在门口 → 黑屏

## 情绪曲线

把全剧每集的情绪强度（0-10）画成曲线，要符合：

- 第 1 集：8（钩子拉满）
- 第 2-3 集：6-7
- 第 4-9 集：5-7（缓冲 + 升）
- 第 10 集（付费点）：9
- 第 11-14 集：6-7
- 第 15 集：8
- ...
- 倒数第 3 集：10（最高）
- 最后一集：8（收尾余韵）

不要让情绪连续 3 集低于 6——观众会弃剧。

## 伏笔管理

每埋一个伏笔，要在 `beat_sheet.json` 标记：

```json
{"t": 30, "type": "setup", "note": "女主桌上那枚旧戒指", "payoff_ep": 18}
```

阶段 5 批判时检查：所有 setup 是否都有对应 payoff。

## 自查清单

写完 `full_script.md` + `beat_sheet.json` 后：

- [ ] 每集都有 cold_open
- [ ] 每集都有 cliffhanger
- [ ] 平均每 20s 有 small_climax / twist
- [ ] 每集 ≥ 1 个 satisfaction（爽剧 ≥ 2）
- [ ] 付费短剧每 5 集有 paywall_punch
- [ ] 情绪曲线符合上述模型
- [ ] 所有 setup 有 payoff
- [ ] 没有连续 3 集情绪 < 6
